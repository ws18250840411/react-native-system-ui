## 💡 分析说明

### 大型组件（≥5KB）
这些组件通常包含复杂的交互逻辑、动画或大量功能：
- **picker** (12.12 KB) - 滚轮选择器，支持多列、级联等复杂场景
- **swiper** (10.86 KB) - 轮播组件，包含手势识别和动画逻辑
- **tabs** (10.53 KB) - 标签页，包含滚动、动画等复杂交互

### 中型组件（2-5KB）
这些组件功能完整，但相对精简：
- **form** (7.02 KB) - 表单验证逻辑
- **cascader** (6.85 KB) - 级联选择器
- **button** (6.55 KB) - 按钮组件（包含 Group）

### 小型组件（<2KB）
这些组件功能简单，体积小：
- **count-down** (1.13 KB) - 倒计时
- **safe-area** (1.45 KB) - 安全区域适配
- **divider** (1.52 KB) - 分割线

## 📝 注意事项

1. **大小单位**：所有大小均为 Gzip 压缩后的字节数
2. **按需导入**：支持 Tree Shaking，实际使用大小 = 引入组件大小之和
3. **依赖关系**：某些组件可能依赖其他组件，实际大小可能略大于单独组件大小
4. **持续优化**：组件大小会随着代码优化而变化，建议定期更新此报告

## 🔄 更新方法

运行以下命令更新组件大小数据：

```bash
npm run build
npm run docs:update-size
```
