### 为什么大小增加了？

1. **版本差异**：
   - 优化前的数据来自较早的版本
   - 在此期间可能添加了新功能、修复了 bug、改进了实现
   - 这些功能增强会导致组件大小增加

2. **代码优化的影响**：
   - 本次优化主要是代码格式和表达式的简化
   - 条件渲染：`{condition ? <Component /> : null}` → `{condition && <Component />}`
   - 样式对象：`...(condition ? { style } : null)` → `...(condition && { style })`
   - 三元运算符格式简化
   - useMemo 依赖数组格式优化

3. **Gzip 压缩影响**：
   - 这些优化主要减少的是源代码行数（约 101+ 行）
   - 但经过 Gzip 压缩后，格式差异的压缩效果很好
   - **代码优化对压缩后大小的影响很小**（通常 < 1%）

4. **实际影响**：
   - **源代码**：减少了约 101+ 行代码（约 3-4 KB 未压缩）
   - **压缩后**：代码优化本身对大小影响很小
   - **可读性**：代码更简洁、更易维护

## 📋 优化前后对比示例

### 示例 1：条件渲染优化

**优化前**（约 3 行）：
```tsx
{loading ? (
  <ActivityIndicator size="small" color={tokens.colors.loading} />
) : null}
```

**优化后**（约 2 行）：
```tsx
{loading && (
  <ActivityIndicator size="small" color={tokens.colors.loading} />
)}
```

**影响**：源代码减少 1 行，但 Gzip 压缩后大小几乎无变化

### 示例 2：样式对象优化

**优化前**：
```tsx
style={[
  baseStyle,
  ...(isActive ? { backgroundColor: tokens.colors.active } : null),
]}
```

**优化后**：
```tsx
style={[
  baseStyle,
  ...(isActive && { backgroundColor: tokens.colors.active }),
]}
```

**影响**：代码更简洁，但压缩后大小几乎无变化
