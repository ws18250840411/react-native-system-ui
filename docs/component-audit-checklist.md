## 审查优先级建议

### 高优先级（核心组件，使用频率高）
1. Button 按钮
2. Input 输入框
3. Form 表单
4. Dialog 弹窗
5. Popup 弹出层
6. Toast 轻提示
7. Loading 加载

### 中优先级（常用组件）
1. Cell 单元格
2. Field 输入项
3. Switch 开关
4. Radio 单选框
5. Picker 选择器
6. ActionSheet 动作面板
7. Avatar 头像

### 低优先级（特殊场景组件）

## 审查流程

1. 按照 [`component-audit-requirements.md`](./component-audit-requirements.md) 的审查清单逐项检查
2. 对照 React Vant 官方文档和源码，确保 API 一致
3. 参考 React Native Paper 的实现细节，确保 RN 端体验一致
4. 检查三端兼容性（iOS、Android、Web）
5. 优化代码，避免过度设计
6. 补充测试覆盖
7. 更新文档
8. 完成后将状态改为 ✅，并填写审查日期和备注
