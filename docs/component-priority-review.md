# 组件优先级审查清单

> 根据组件复杂度、风险点、影响面等因素，列出需要逐个对齐检查的重点组件。

## 审查标准

- **复杂度指标**：手势处理、动画、多端差异、状态管理复杂度
- **风险指标**：回归频率、用户影响面、测试覆盖度
- **代码质量**：代码行数、逻辑分支、性能敏感度

---

## 🔴 P0 - 最高优先级（基础设施 + 高风险）

这些组件影响面大或风险极高，需要优先审查。

| 组件 | 复杂度 | 主要风险点 | 审查重点 |
|------|--------|-----------|---------|
| **overlay** | ⭐⭐⭐⭐⭐ | 栈管理/zIndex/Portal/遮罩 | 1. OverlayStack 栈管理逻辑<br>2. zIndex 分配策略<br>3. Web/Native Portal 差异<br>4. 遮罩点击/滚动锁定 |
| **portal** | ⭐⭐⭐⭐⭐ | 渲染/挂载/回收/快照 | 1. Web 自动挂载/回收<br>2. Native PortalHost 管理<br>3. 快照缓存机制<br>4. zIndex 透传 |
| **popup** | ⭐⭐⭐⭐ | 弹层/定位/动画/安全区 | 1. 关闭链路去重<br>2. 布局测量等待<br>3. 动画/安全区稳定<br>4. 定位计算（measureInWindow） |
| **swiper** | ⭐⭐⭐⭐⭐ | 滑动/循环/指示器/虚拟化 | 1. Web/Native 滚动差异<br>2. 循环模式边界处理<br>3. Ref 跳转逻辑（同页不锁死）<br>4. 滚动队列管理<br>5. 虚拟化性能 |
| **tabs** | ⭐⭐⭐⭐⭐ | swipeable/指示器/滚动/测量 | 1. 滑动切换逻辑<br>2. 指示器动画<br>3. 滚动测量与定位<br>4. 嵌套滚动冲突 |

---

这些组件涉及复杂手势或交互，容易产生回归。

| 组件 | 复杂度 | 主要风险点 | 审查重点 |
|------|--------|-----------|---------|

| **swipe-cell** | ⭐⭐⭐⭐⭐ | 滑动手势/点击回收/滚动冲突 | 1. 左右滑动手势识别<br>2. 点击回收逻辑<br>3. 与列表滚动冲突<br>4. 多实例互斥 |
| **pull-refresh** | ⭐⭐⭐⭐ | Web 手势/状态派生 | 1. Web 拖拽手势模拟<br>2. 状态派生逻辑<br>3. 受控/非受控刷新<br>4. 回调与布局计算 |
| **index-bar** | ⭐⭐⭐⭐ | 手势拾取/滚动联动 | 1. PanResponder 手势拾取<br>2. 滚动联动逻辑<br>3. 指示器显示<br>4. 锚点定位 |
| **picker** | ⭐⭐⭐⭐ | 级联/虚拟渲染/滚动 | 1. 级联选择逻辑（已修复索引比较问题）<br>2. 虚拟渲染优化<br>3. Web/Native 滚动差异<br>4. 值归一化处理 |

| **number-keyboard** | ⭐⭐⭐ | 弹层+输入联动 | 1. 弹层显示/隐藏<br>2. 输入联动逻辑<br>3. 交互细节稳定<br>4. 回调/渲染节点最小化 |

## 审查检查清单（每个组件）

### 1. 代码质量
- [ ] 是否使用 `React.useMemo` / `React.useCallback` 优化
- [ ] 是否有不必要的重渲染
- [ ] 代码是否足够精简（无冗余逻辑）
- [ ] 是否有 `any` 类型（应避免）

### 2. 多端兼容性
- [ ] Web/Native 事件处理差异
- [ ] 动画 Native Driver 使用（Web 禁用）
- [ ] 样式兼容性（Hairline、fixed 布局等）
- [ ] 手势处理（Web 鼠标/触摸，Native PanResponder）

### 3. 性能优化
- [ ] 虚拟化（长列表）
- [ ] 动画性能（Native Driver）
- [ ] 布局测量优化（避免频繁测量）
- [ ] 状态更新优化（避免不必要的更新）

### 4. 测试覆盖
- [ ] 核心逻辑单测
- [ ] 边界值测试
- [ ] 多端兼容性测试
- [ ] 交互测试（手势、动画）

### 5. 文档完整性
- [ ] API 文档完整
- [ ] Demo 示例充足
- [ ] 多端差异说明
- [ ] 性能注意事项

---

## 建议审查顺序

1. **第一轮（P0）**：overlay → portal → popup → swiper → tabs
2. **第二轮（P1）**：slider → swipe-cell → pull-refresh → index-bar → picker
3. **第三轮（P2）**：skeleton → stepper → input → password-input → number-keyboard
4. **第四轮（P3）**：toast → dialog → action-sheet → share-sheet → uploader
5. **第五轮（P4）**：tabbar → calendar → datetime-picker → form

---

## 审查输出

每个组件审查后应输出：
1. **问题清单**：发现的问题及修复建议
2. **优化建议**：性能、代码质量优化点
3. **测试补充**：需要补充的测试用例
4. **文档更新**：需要更新的文档内容
