## 优化重点

### 1. API 表格优化

**优化前**：
```markdown
| `type` | 类型，可选值为 `default` `primary` `info` `warning` `danger` `success` | `ButtonType` | `default` |
```

**优化后**：
```markdown
| `type` | 按钮类型 | `'default' \| 'primary' \| 'info' \| 'warning' \| 'danger' \| 'success'` | `'default'` |
```

**原则**：
- 说明简洁，一行描述
- 类型明确，使用字面量类型
- 默认值用引号包裹字符串

### 2. 介绍部分优化

**优化前**：
```markdown
## 介绍

按钮用于触发一个操作，如提交表单。按钮是用户界面中最常用的交互元素之一，它可以让用户执行各种操作，包括提交表单、确认操作、取消操作等。
```

**优化后**：
```markdown
## 介绍

按钮用于触发操作，如提交表单。
```

**原则**：
- 一句话说明用途
- 不超过 2 行
- 避免冗余描述

### 3. 说明文字优化

**优化前**：
```markdown
通过 `plain` 属性将按钮设置为朴素按钮，朴素按钮的文字为按钮颜色，背景为白色。
```

**优化后**：
```markdown
通过 `plain` 属性设置为朴素按钮，文字为按钮颜色，背景为白色。
```

**原则**：
- 去除重复词汇
- 使用简洁表达
- 保留关键信息
