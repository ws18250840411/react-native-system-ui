# 组件文档模板

> 本文档定义了组件文档的标准格式，所有组件文档必须遵循此模板。

## 文档结构

```markdown
---
simulator:
  compact: false  # 或 true，根据组件特性决定
---

# ComponentName 组件名称

## 介绍

简洁明了的一句话介绍组件用途，不超过 2 行。

> 可选：重要提示信息，如特殊用法、注意事项等。

import { ComponentName } from 'react-native-system-ui'
```

## 编写规范

### 1. 介绍部分
- **必须**：一句话说明组件用途
- **可选**：重要提示用 `>` 引用块
- **禁止**：冗长的背景介绍

### 2. 代码演示部分
- **标题**：简洁明了，如"基础用法"、"自定义样式"、"禁用状态"
- **Demo 文件**：统一放在 `demo/` 目录下
- **命名**：使用 kebab-case，如 `basic.tsx`、`custom-style.tsx`

### 3. API 表格
- **属性说明**：一行描述，不超过 20 字
- **类型**：使用 TypeScript 类型，如 `'top' \| 'bottom'`
- **默认值**：明确标注，无默认值用 `-`
- **排序**：按重要性排序，常用属性在前

### 4. 说明文字
- **简洁**：避免冗余描述
- **准确**：与代码实现一致
- **易懂**：使用通俗易懂的语言

### 5. 差异说明
- **仅在有差异时**：与参考库有明显差异时才说明
- **明确标注**：说明差异原因和影响
